# FocusDo 产品分析

## 一、当前产品现状

### 核心功能清单

| 功能 | 状态 | 评价 |
|------|------|------|
| 三列看板（投资/工作/生活） | ✅ | 核心骨架 OK |
| 今日焦点面板 | ✅ | 差异化亮点 |
| 筛选（全部/进行中/今日/已完成） | ✅ | 基本够用 |
| 快速添加（分类/优先级/日期/今日标记） | ✅ | 信息密度合理 |
| 进度环 | ✅ | 视觉反馈不错 |
| 勾选完成/删除 | ✅ | 有微动效 |

---

## 二、致命问题（不修就不能用）

### 1. 零持久化 —— 刷新即失
所有数据在内存里，刷新页面回到初始状态。用户添加的任何东西都会丢失。P0 级别问题。

### 2. 没有编辑能力
创建之后就不能改了。发现打错字？只能删了重建。没有编辑文本、改优先级、改日期的功能。

### 3. 没有 undo / 删除确认
点了删除就没了，没有二次确认，没有撤销提示。误删 = 数据永久丢失。

---

## 三、体验短板（用了会觉得不好用）

### 4. 没有搜索
当 todos 多了之后，找特定事项只能靠眼睛一个个扫。至少需要一个文本搜索。

### 5. 列不能自定义
三列（投资/工作/生活）是写死的。用户可能想加一列"学习"、减掉"投资"，或者改名字/改颜色。没有增删改项目的能力。

### 6. 没有排序
todos 的顺序是创建顺序。不能按优先级、截止日期、完成状态排序。

### 7. 筛选 Tab 不够直观
"全部"和"进行中"在语义上很接近，用户要思考才能选对。可以改成更清晰的分类。

---

## 四、缺失的差异化功能（做了就能拉开差距）

### 8. 拖拽排序 / 跨列移动
看板类产品没有拖拽是不完整的。至少支持同列内拖拽排序，最好支持跨列移动。

### 9. 每日回顾 / 周报生成
既然有"今日焦点"的概念，应该有"今日回顾"——自动汇总今天完成了什么、逾期了什么，生成一段文字摘要。这是 FocusDo 跟普通 Todo 最大的差异点。

### 10. 数据统计仪表盘
环形进度只在列头有一个小的。可以做一个汇总页：本周完成率、各项目对比、逾期趋势、完成热力图。

### 11. 快捷键支持
`N` 新建、`/` 搜索、`1/2/3` 切换项目列、`Enter` 确认、`Esc` 取消——对效率用户是刚需。

### 12. 暗色模式
现在是暖色纸张风格，暗色模式需要单独适配。

## 五、迭代路线图

| 阶段 | 目标 | 具体事项 |
|------|------|----------|
| **v0.1 — 能用** | 不丢失数据 | LocalStorage 持久化、编辑 todo、删除确认 |
| **v0.2 — 好用** | 核心体验闭环 | 搜索、排序、自定义项目列、undo |
| **v0.3 — 想用** | 差异化 | 拖拽排序、每日回顾摘要、快捷键 |
| **v0.4 — 分享** | 增长 | 数据统计仪表盘、暗色模式、导出/分享 |

## 六、技术债务

- 所有样式使用 inline style，维护成本高，后续建议迁移到 CSS Modules 或 Tailwind
- 数据类型定义在组件文件内，应抽离到独立的 types 文件
- 没有单元测试和 E2E 测试
- `nextId` 使用模块级变量，多实例不安全，应迁移到 UUID 或 crypto.randomUUID()
